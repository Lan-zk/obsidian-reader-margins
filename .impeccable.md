# Impeccable Design Context

## Design Context

### Users
Obsidian 桌面端重度用户：用内置 PDF 阅读器做深度阅读与知识管理的人（研究者、学生、写作者）。使用场景是**长时间专注阅读**——插件界面是阅读体验的旁注层，不是主角。他们要完成的事：选中 PDF 正文 → 高亮/下划线 → 在页边距写批注 → 卡片可拖动/换色/删除 → 导出 Markdown。

### Brand Personality
- 基础人格：**Atomic Minimalism**（docs/design.md）——扁平、无阴影、1px 边框、色调分层、4px/8px 圆角、Hanken Grotesk、克制低饱和调色板（Steel Blue 主色，Muted 语义色），明暗双主题。
- 三个词：专业、可靠、有手感。
- **2026-07 用户决定的演进方向**：在 Atomic Minimalism 基础上**明显更大胆**——允许更强的色彩运用、更丰富的层次、更有冲击力的交互状态，可以偏离"quiet/invisible"的原始定位，但不能牺牲长时间阅读的舒适度。
- Anti-references：通用 AI 渐变大字标题、玻璃拟态、发光暗色面板、模板化卡片网格、装饰性阴影堆叠。

### Aesthetic Direction
- 明暗双主题都必须高质量支持（Obsidian 跟随用户主题）。
- 颜色：标注色（默认黄/蓝/绿/红，可配置）是语义色，用于状态与标识；在更大胆方向下，标注色可以承担更强的视觉角色（选中态 tint、边框、连接线强调）。
- 排版：Hanken Grotesk 全家族；label 级别允许大写+加字距。
- 圆角语言 "Soft-Square"：4px 基础，卡片 8px 上限。
- 扁平优先：阴影和模糊仍是禁忌；层次靠 1px 边框 + 色调分层表达。

### Design Principles
1. **阅读第一**：批注层永远服务于正文；任何动效与视觉增强都不能抢占阅读注意力（动效以状态反馈为主，时长短、缓动为指数衰减）。
2. **状态即色彩**：标注色承载选中/悬停/链接状态；大胆体现在色彩运用强度，而非新增装饰元素。
3. **4px 原子网格**：所有间距、内边距必须是 4 的倍数；对齐优先于宽松。
4. **1px 边框 + 色调分层**：不用阴影/模糊表达深度。
5. **无障碍底线**：`prefers-reduced-motion` 下所有动效降级为静态状态；色彩对比度满足 WCAG AA（正文文本），色盲用户不靠颜色唯一线索（图标/形状双编码）。
6. **覆盖范围**：优化作用于全部插件界面——批注卡片、连接线、高亮/下划线、工具栏、设置面板、导出 UI。
